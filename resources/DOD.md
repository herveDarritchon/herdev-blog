### Definition of Done (DoD) — Minimal vs Complet

#### DoD MINIMAL (FICHE / CHEAT SHEET)
Une fiche est “DONE” si :
- **Intention** : 1 problème précis (symptôme + objectif) en 1–2 phrases.
- **Fix** : 1 procédure courte (étapes) sans sauts logiques.
- **Validation** : 2–5 checks observables (ce qui prouve que c’est bon).
- **Vérité** : tout point non trivial est taggé **[Vérifié doc]** (lien) ou **[À vérifier]** (commande) ou **[Hypothèse]** (méthode).
- **Contexte** : OS/shell/outils/versions, ou “À reproduire” + commande(s) de collecte versions.
- **Références** : au moins 1 lien primaire (doc officielle / release notes).
- **Meta** : TL;DR + `<!--more-->` + tags autorisés. (Changelog optionnel sur une fiche.)

#### DoD COMPLET (TUTORIEL / GUIDE / CADRE / REX)
Un post est “DONE” si :
- **Problème & contexte** : intention + persona + contexte (contraintes) explicites.
- **Prérequis** : versions/outils/OS/shell (ou “À reproduire” + collecte versions).
- **Procédure** : étapes numérotées + checkpoints (tuto/guide) ou critères + options + décision (cadre).
- **Alternatives & trade-offs** : ≥2 options sérieuses + critères de choix + limites.
- **Pièges** : erreurs fréquentes + prévention.
- **Validation** : checklist de sortie complète (signaux de succès + anti-signaux).
- **Sources** : doc primaire + références complémentaires si utile (pas de blabla SEO).
- **Maintenance** : section “Last updated” + **Changelog** (au moins 1 entrée quand c’est mis à jour).
- **Maillage** : si satellite → lien vers 1 pilier pertinent ; si pilier → liens vers satellites pertinents.
- **Meta** : TL;DR + `<!--more-->` + tags autorisés.
