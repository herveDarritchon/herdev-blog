Cette page explique clairement les licences applicables au contenu de ce blog (textes, images), aux extraits de code
publiés dans les articles, au thème, et aux contenus tiers.

> Disclaimer : je ne suis pas avocat. Ceci décrit mes choix et mon intention, pas un conseil juridique.

## Résumé

- **Articles (texte)** : **CC BY-NC-ND 4.0** *(Attribution • Pas d’usage commercial • Pas de modifications)*
- **Images originales** : **CC BY-NC-ND 4.0** (sauf mention contraire sous l’image)
- **Extraits de code dans les articles** : **MIT**
- **Thème (code)** : **MIT**
- **Contenus tiers** (logos, screenshots, polices, etc.) : **leur licence**

## 1) Licence du contenu éditorial (textes)

Sauf mention contraire, mes articles sont sous licence :

**Creative Commons Attribution-NonCommercial-NoDerivatives 4.0 International (CC BY-NC-ND 4.0)**

### Tu peux
- partager un lien vers un article ;
- citer des **extraits courts** avec attribution ;
- repartager l’article **tel quel** (sans le modifier), avec attribution, et sans usage commercial.

### Tu ne peux pas (sans autorisation)
- republier l’article en le **modifiant** (même “un peu”) ;
- le **traduire** et le publier (une traduction = une œuvre dérivée) ;
- en faire une version “résumée/rewritten” publiée ailleurs ;
- l’utiliser dans un produit/service payant, newsletter premium, formation payante, etc.

> Oui, c’est strict. C’est assumé : je veux encourager le partage, pas l’appropriation ni les réécritures sans contrôle.

## 2) Images : attention, c’est le piège classique

Les images ne sont pas toutes égales.

### Images que j’ai créées
Par défaut : **CC BY-NC-ND 4.0** (comme les textes).

### Images / logos / marques de tiers
- Les marques, logos, screenshots d’outils, captures de docs, etc. restent **la propriété de leurs auteurs**.
- Si j’en utilise, c’est dans un cadre de démonstration, critique, illustration, ou citation.
- **Tu ne peux pas les réutiliser automatiquement** juste parce qu’elles sont sur mon blog.

> Si tu veux que ce soit clean : ajoute une ligne de crédit sous chaque image “tierce” (source + licence).

## 3) Snippets de code dans les articles (licence MIT)

Les blocs de code destinés à être copiés/collés sont sous licence **MIT**.

Pourquoi : parce que **CC BY-NC-ND est toxique pour le code** (ND bloque les adaptations, NC bloque les usages pro). Résultat : personne n’ose réutiliser, et c’est l’inverse de l’esprit dev.

Concrètement :
- tu peux réutiliser et adapter ces snippets, même en contexte commercial ;
- tu gardes la mention MIT si tu republies le code tel quel.

> Ça ne veut pas dire que *tout le repo du blog* est MIT. Ça veut dire : **les snippets “code” dans les posts**.

## 4) Licence du thème (code) : MIT

Le thème que j’utilise et maintiens est du **code**. Il doit rester **open-source et réutilisable**, sinon tu te tires une balle dans le pied :
- pas de contributions,
- pas d’adoption,
- pas d’écosystème.

Donc : **le thème est sous MIT**.

## 5) Polices, assets, dépendances : chacun sa licence

J’héberge certaines polices et assets pour des raisons de performance / contrôle (pas de dépendance runtime à un CDN).
Mais :
- **une police = une licence** (souvent OFL ou équivalent),
- **un framework CSS = une licence**,
- **une icône = une licence**.

Si tu réutilises ce blog ou ce thème, tu dois vérifier les licences de chaque asset tiers (polices, images, etc.) et respecter leurs conditions.

## 6) Demander une autorisation

Si tu veux :
- traduire un article,
- republier une version modifiée,
- utiliser un contenu en contexte commercial,
- réutiliser une image spécifique,

contacte-moi : je réponds vite quand la demande est claire (lien vers le contenu + usage prévu + où + combien de temps).
